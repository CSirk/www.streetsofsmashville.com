C# : Classes

C# is an Object Oriented Programming Language, meaning the majority of things you will do will involve objects, which are used to model data. A class is a grouping of properties, methods, and other data which allows you to create objects with the class as a blueprint. For example. Lets say that you would like to model a person in code, a class is the way to do that. A Person has attributes associated with them such as number of eyes, number of legs, a name, age, and actions they can performs such as walking or swimming. A class can put this person with these attributes into your program and you can interact with the person as an object. Rather than talk about how this is possible in words, lets build an appliction and insert this person into it!


Note: Keywords are written in this style and code is written in this style

  1. Start Visual Studio
  2. Go to File > New Project > C# Console Application
  3. Name the solution and project anything you like (for example: ClassesTutorial) and save. (This will generate your project and load Program.cs
  4. Right click on your project and go to Project > Add > New Item > Class and name this class Person. Press ok to generate the new class file. This should load the new class onto your screen.
  5. Make the following change to your class's code:

    6. add the word public in front of the class name.

  6. Add the following lines to your code file:

    7. public string Name { get; set; }

    public int Age { get; set; }

    public int NumberOfLegs { get; set; }

    public int NumberOfEyes { get; set; }

    public int Walk(int numberOfLegs, int steps);

    public void Swim();

  7. Go back to your Program.cs file and inside the Main method and type:

    8. Person person1 = new Person();

  8. Under this line of code, type the following:

    9. person1.Name = "Cody";

    person1.Age = 24;

    person1.NumberOfLegs = 2;

    person1.NumberOfEyes = 2;

    So far we have made an object of type "Person" and given it a name, age, and told the number of eyes and legs the person has, but we dont have anything to show for it! We need to print to the screen to get some meaningful output about the person.

  9. Under the code you just wrote, write the line:

    10. Console.WriteLine($"Hello, I am {person1.Name}. I am {person1.Age}. How many eyes and legs do i have? What a weird question.. obviously I have {person1.NumberOfLegs} legs and {person1.NumberOfEyes} eyes.");

    Console.ReadKey();

    Alright! So now when you run the appliction you will learn a little bit of information about your person you created! Lets make this a little more interesting for our person, after all.. he did call us weird for asking about how many legs and eyes he has. Lets give him a little competition and see how he stacks up against another person we create! First lets make the creation of another person a little easier on ourselves by making a constructor for the class so we can feed the object data as it is created.

  10. Go back the the Person.cs class and type the following above the properties and methods we added:

    11. public Person() { }

    public Person(string name, int age, int numberOfLegs, int numberOfEyes = 2)

    {

    this.Name = name;

    this.Age = age;

    this.NumberOfLegs = numberOfLegs;

    this.NumberOfEyes = numberOfEyes

    }

    This constructor will allow us to pass in the values for the person without having to do so line by line. There are many ways to hydrate an object of a class and we will go into a few more later in the tutorial.

  11. Go back to Main.cs and type the following after the declaration of the last person variable

    12. var person2 = new Person("BetterCody", person1.age + 1, person1.NumberOfLegs + 3, person1.NumberOfEyes + 50);

    We have done a little constructor magic here and have created a person who is better than cody in every way. Notice that we took cody's (our person1 variable) and added to each property to pass values to the constructor. It would have been just as easy to instantiate the new person with the code line: var person2 = new Person("BetterCody", 25, 3, 3); , but the way we did it allows the new and improved cody to always be better than the old cody because we use his own values and then add to them: if cody's number of eyes changes so does our better Cody's number of eyes.. no matter what cody does better cody will always be better. darn. Now i feel bad. Lets make another person who will lose to cody in our contest in a few minutes. This time we will declare him in a different way than with a constructor or line by line, we will use an inline declaration.

  12. Type the following into main.cs after person2 is declared.

    13. var lesserCody = new Person { Name = "Lesser Cody", Age = 3, NumberOfLegs = 0, NumberOfEyes = 1 };

  13. Type the following:

    14. Console.WriteLine("Cody, I challenge you to a race.");

    Console.WriteLine($"{person1.Name} : who said that?");

    Console.WriteLine($"{person2.Name} : I did.");

    Console.WriteLine($"{lesserCody.Name} : I also want to participate in the race.");

    Console.WriteLine("Press any key to start the race.");

    Console.ReadKey();

    Console.WriteLine($"");
